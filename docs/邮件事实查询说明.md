# 邮件事实查询说明

## 定位

Mail Facts Service 是统一邮件归档之上的只读事实层，供后续 GUI、诊断和受控内部调用读取邮件、资源与会话。它不提供发送、删除、移动、标记、执行附件、访问链接或修改归档的能力，也不承担 Hermes、Obsidian 或知识管理职责。

## 可用查询

- `list_mail_messages`：分页列出邮件，可按账号、邮箱目录、日期、发件人、主题、是否含附件和归档状态过滤。
- `get_mail_message`：读取一个 package 的元数据、正文事实、计数、结构化联系人、outbound origin 和归档状态。
- `list_mail_resources`：列出某封邮件的正文层、内嵌图片、附件、链接和下载资源。
- `list_mail_threads`：按 `thread_ref` 聚合可确认的会话；无法确认的邮件保持独立。
- `get_mail_thread`：读取一个会话内按时间排序的邮件。
- `search_mail_facts`：搜索主题、发件人显示名/地址、收件人、抄送人、完整可读正文、附件/邮件图片显示名与原始名、链接文字、域名、完整 URL、资源状态和邮件自然状态。

ApplicationService 与 MCP 的 `search_mails` 在此基础上增加 latest/today/yesterday/recent_days/date_range/all 时间语义、newest/oldest 排序、结构化过滤、分页以及可选 `ensure_fresh`。空 query 仍可与结构化过滤组合，不会退化为无界扫描。

v1.4.3 中，指定 `account_id` 的 `ensure_fresh` 通过 Account Runtime Router 同步同一账号，并使用该账号独立凭据/OAuth、进程锁、重试和调度状态；Gmail 使用既有 API/IMAP Adapter，QQ、163 与 Generic 使用共享 Generic IMAP Core。省略账号仍保留兼容当前收件账号语义；未启用 receive 的账号会明确拒绝。

所有方法返回可 JSON 序列化的字典与列表。默认有界分页，`limit` 和 `offset` 必须受服务层约束，避免一次加载全部邮箱。

## 过滤与搜索语义

账号优先使用稳定 `account_id`，`account_ref` 只供 v1.3 客户端兼容；邮箱范围同时保留正式 `mailbox_id` 与 Provider `mailbox_ref`。省略账号时返回统一跨账号视图，指定账号时邮件、会话和资源命中严格隔离。日期过滤基于已保存的发送/接收事实；发件人和主题使用大小写不敏感匹配；“含附件”依据权威 package 计数，不从历史行或文件名推断。

搜索文本由主题、解码后的 From/To/Cc/Bcc/Reply-To 显示名与地址、可读正文和资源显示事实组成。HTML 脚本与样式不会进入可读正文。普通 URL 和其中的域名、路径、查询片段可以命中，但查询不会访问该 URL。连续空白会归一化为多个关键词并使用 AND 语义；每个关键词可命中不同邮件事实。查询从 `mail_packages` 返回唯一邮件行，资源使用相关子查询，因此同一邮件有多个附件或正文与链接同时命中都不会重复。

GUI 收件页继续保持“今天”范围，并在输入停止约 250ms 后调用事实搜索；空查询恢复当天 `list_mail_messages`。底层方法仍接受账号、邮箱目录和日期范围，不把“今天”写死，便于未来受控调用复用。

## 返回边界

用户可见分类使用中文产品文案，例如正文、邮件中的图片、附件、网页链接、可下载文件链接、云文档、外部图片和已下载文件。链接名称会结合类型、provider、hostname、路径和有意义的 anchor；完整 URL 位于详情、Tooltip 和显式操作。内部 `resource_type/source_type` 只在结构化详情需要时保留。

联系人旧字段继续返回人类可读值，并增加 `from_display`、`from_address`、`from_addresses`、`to_addresses`、`cc_addresses`、`bcc_addresses` 和 `reply_to`。原始 encoded Header 集中在 `raw_headers`，`raw.eml` 保持原样；GUI 和 MCP 的普通展示字段不回退为 encoded-word。

查询返回 raw 的路径、状态、大小和 Hash 事实，但不返回 `raw.eml` 字节；也不读取或回显 Credential Manager、OAuth token、邮箱密码、QQ 授权码或私人配置。

本地文件路径用于受控打开、复制和一致性诊断。主表是否展示路径由 GUI 信息架构决定，事实层不会截断或伪造路径。

## 权威来源

- 邮件身份、正文事实、资源计数与归档状态：`mail_packages`。
- 账号、能力、数据命名空间与邮箱目录：`mail_accounts`、`mailboxes`。
- 正文层、图片、附件、链接和下载资源：`mail_resources`。
- 当前业务历史兼容：`received_messages`。
- 当前文件管理兼容：`received_files`。

兼容表可以帮助现有页面运行，但不得反向覆盖权威邮件事实。GUI 与 MCP 都优先通过 ApplicationService 的 Mail Facts 方法读取。MCP 读取开关只控制 Agent 入口，不改变事实层只读性质，也不授予 Gmail 修改权限。

## 会话安全退化

Gmail 有可靠 thread id 时直接使用。IMAP 只有 References/In-Reply-To 时建立关系；缺少可靠关系时允许 `thread_ref` 为空或退化为 Message-ID，绝不按相似主题、发件人或时间窗口猜测并合并邮件。
