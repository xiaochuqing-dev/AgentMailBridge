# AgentMailBridge GUI 使用说明

AgentMailBridge 1.0.0 的用户入口只有 `AgentMailBridge.exe`。

顶部主工作区只有“收件”和“发件”。左侧已有 Gmail、QQ 账号卡片用于管理当前账号；“添加邮箱账号”只展示未来扩展和当前限制，不会修改已有账号。左侧底部只有“历史记录”“文件与数据”“设置”“关于”。

收件页已经升级为邮件级界面。“今日收到邮件”以一封邮件一行展示主题、发件人、内容摘要、时间、状态和操作；多个附件不会拆成多行。双击邮件或点击“查看”进入邮件详情，正文以安全可读文本呈现，内嵌图片、附件和链接按业务名称分组，并可继续查看同一会话。页面不向普通用户显示 package、resource_kind 等内部术语。

- 收件：标题右侧唯一刷新、自动收取、检查间隔、连接测试、立即收取、可编辑收件偏好、今日文件和最近日志。Gmail 管理只保留在左侧账号卡，不在收件页重复显示。
- 发件：编辑可选主题、正文、0 至多个附件和 0 至多个链接，一次确认发送一封邮件；附件继续执行快照与 Hash 保护，收件人固定。最近发送一封邮件一行，Agent / MCP 从标题区进入。
- 历史记录：以“类型、摘要、时间、状态、操作”查看收件、发件、Agent / MCP 业务记录，可按关键词或 request_id 搜索并定位关联文件；request_id、完整路径、原始状态和错误信息只在结构化详情中显示。
- 文件与数据：通过统一受管文件数据源管理收件文件、发送归档和去重后的 Agent 结果，并提供文件详情、存储概览、备份、恢复和一致性扫描。
- 设置：开机启动、界面主题、单次收取量和最大发送文件大小。
- 设置 > 高级设置：网络模式、Runtime Paths、配置迁移和脱敏高级诊断。
- 关于：版本、产品定位、仓库、LICENSE、第三方说明和本地优先说明。

Gmail API、Gmail IMAP 和 QQ SMTP 分别使用专属账号页。Gmail IMAP 密码和 QQ 授权码保存在 Windows Credential Manager，界面只显示固定掩码。替换 OAuth 客户端配置需确认，不会静默删除 token。

“当前收件偏好”默认是“仅本人邮件”。点击“编辑偏好”可选择“仅本人邮件”“当前扫描范围内全部邮件”或“自定义规则”。自定义规则只支持发件人完整邮箱或 `@domain.com`、主题关键词和“仅保存含附件的邮件”；不同分类必须同时满足，同一分类命中任一项即可。空自定义规则不能保存。该设置由 Gmail API、Gmail IMAP、手动立即收取和自动收取共用，旧布尔配置会自动无损迁移。

收件提示分为四类：成功为绿色；正常检查但无新邮件为中性提示；部分完成为黄色警告；主任务无法执行才显示红色失败。无新邮件不会增加失败统计。

自动收件默认每 60 秒检查一次，最低 30 秒。开启或应用启动后约 3 秒首次检查；收件页实时显示正常运行、正在检查或连接退避，以及上次检查、上次成功、下次检查、最近结果和待重试邮件。页面刷新和应用重启后状态从 SQLite 恢复。

Gmail API 与 IMAP 都会重叠回看最近约 30 分钟并去重。到期坏邮件即使离开回看窗口，仍按 Gmail 资源 ID 或 IMAP UID 单独重试。网络、授权或后端整体故障会自动退避，恢复成功后回到正常周期；系统睡眠、长暂停或事件循环恢复后会补偿检查。单封坏邮件会有限重试并最终标为需要处理，不影响后续邮件。“立即收取”始终保留，但与自动收取共用互斥锁，正在检查时不会启动第二个任务。

今日邮件使用“主题、发件人、内容、收取时间、状态、操作”六列。主题和内容按可读宽度换行并自动增加行高，不使用生成的省略号隐藏核心信息；操作按钮保持可点击和垂直居中。

“文件与数据”中的新收件文件来自邮件资源表，未映射的旧 `received_files` 记录继续兼容；发送归档来自 `sent_files`，Agent / MCP 结果与对应发件按关联关系去重。主表增加“所属邮件”，支持从文件进入邮件详情，邮件详情也可定位附件。主表不展示绝对路径，完整路径位于文件详情和复制操作。大小显示区分 `0 B`、未知的 `—` 和“文件已不存在”。

发件正文、附件和链接均可为空，但四者不能同时为空。附件支持添加、移除和同路径去重；同名不同路径附件仍可同时发送。发送前会显示主题、正文摘要、附件数量、链接数量和固定收件人，确认后只调用一次 SMTP。

“Agent 发件 / MCP”提供完整交付指令和允许工作区管理。添加工作区时会拒绝过宽或敏感目录；授权在下一次 Agent/MCP 进程启动时生效。Agent 应自行定位最终交付文件，使用稳定 request_id 直接提交原始路径，不运行 Copy-Item、cp 或临时搬运；path_not_allowed 时应提示用户在界面授权真实工作区。

右侧“连接健康”分别展示 Gmail 收件、QQ SMTP、Agent/MCP、凭据/OAuth、SQLite/数据目录的状态、说明和最近检查时间；“一键检查全部”刷新五项结果，异常项可直接“去处理”。

正常窗口下，收件和发件核心区域合理分配高度；最大化、还原和手动缩放后继续可用，较矮窗口启用页面滚动兜底。数据较多时各表格内部滚动，不使用分页。完整业务记录由历史记录管理，实际文件和维护操作由文件与数据管理。

关闭主窗口默认进入托盘；托盘可恢复或正常退出。开机启动默认关闭，启用后只启动主程序；内部 MCP 不显示 GUI、托盘或开机启动项。

窗口首次使用正常尺寸启动，只保存 normal geometry，并约束在当前显示器可用工作区内。标题栏最大化按钮在最大化后切换为还原图标；双击标题栏同样切换。关闭到托盘不会停止自动收件，只有托盘菜单“退出程序”才停止调度器。
