# AgentMailBridge GUI 使用说明

## 启动与退出

使用 `python -m agent_mail_bridge.gui` 启动。窗口关闭按钮默认隐藏到系统托盘；需要彻底退出时使用托盘菜单“退出程序”。程序会等待当前后台任务安全结束，并保存窗口位置、大小、上次页面、主题和自动收件设置。

如果上次异常退出，重新启动后会显示提示。临时错误、未确认发送任务和上次待发送文件不会恢复。

## 收件

基础配置页选择 Gmail API、IMAP 或自动模式。自动收件开启时，所有手动收件入口都会禁用；关闭后恢复。手动收件和自动收件都显示执行中、成功、部分成功、失败或需要授权状态。

收件完成后，今日文件、历史、日志、统计和服务状态会自动刷新。今日文件和收件页支持双击安全预览；危险或不支持的类型只在资源管理器中定位，不会执行。

## 发件与文件确认

发件页默认没有待发送文件，发送按钮处于禁用状态。每次发送必须重新点击“选择文件”，可浏览桌面、下载目录、其他磁盘和普通用户目录。选择后请核对文件名、大小、类型、最后修改时间和完整路径。

界面提供复制路径、打开所在文件夹和安全预览。点击发送后还会显示附件名、大小、邮件主题和固定收件人。只有确认后才连接 QQ SMTP。

选择时记录 SHA-256，后台发送前再次检查文件并创建 DATA_ROOT 内的受控快照，快照大小和 Hash 必须与源文件一致。SMTP 和 sent 归档使用稳定快照，邮件附件保留原文件名。发送成功、重复请求或邮件已发出但本地归档失败后，界面会清空本次选择。

GUI 的全局选择属于用户明确授权。Agent 接口和 CLI 仍只能提交 DATA_ROOT / ALLOWED_SEND_ROOTS 内的文件。

## 日志与刷新

首页最近日志和日志页均按最新到最早显示。首页、收件页、日志页和 Agent 接口页都有刷新入口。刷新在线程池中执行，按钮显示“刷新中”，完成后显示最后刷新时间。

## 高级设置与诊断

高级设置包含 QQ SMTP、Gmail 网络模式、收件数量、发送大小和开机启动。Gmail IMAP 密码和 QQ 授权码保存到 Windows Credential Manager，输入框不回显已保存旧值，只显示已配置状态。可更新、删除或测试连接；删除前必须确认。

“临时显示密码和授权码”只改变当前界面显示，不会复制或导出秘密。Gmail API 显式授权、诊断 IMAP、诊断 Gmail API 和诊断 QQ SMTP 都有独立按钮反馈。失败时显示普通用户可理解的建议，“查看最近错误详情”显示已脱敏技术信息。

“导出脱敏诊断报告”可保存 Markdown 报告。目标文件已存在时不会覆盖，需要另选文件名。

旧版 `.env` 用户可运行 `python -m agent_mail_bridge migrate-credentials`。只有安全存储写入并回读成功的项目才会从 `.env` 清空，失败项继续保留。

## 数据维护

“数据维护”页显示 SQLite 完整性、数据库大小、表记录数以及 received、send、sent 和日志占用。可以创建并验证数据库备份、选择备份恢复、执行一致性扫描、打开备份目录和导出脱敏维护报告。

恢复前会验证备份并自动备份当前数据库，运行任务时禁止恢复。恢复只处理数据库，不覆盖 received 或 sent 文件。一致性扫描只报告缺失、孤立、Hash 异常、越界路径和暂存残留，不自动删除数据。

性能基准命令为 `python -m agent_mail_bridge stability-benchmark --records 10000 --cycles 50 --output test-output/stability.json`，使用独立临时数据目录，不污染真实数据。

## 主题、字体与 DPI

标题栏月亮或太阳按钮切换浅色和深色主题。GUI 优先使用 Microsoft YaHei UI，并由 Qt 6 原生处理高 DPI。已检查 100%、1.25 和 1.50 比例；若仍模糊，请清除自行设置的 `QT_SCALE_FACTOR`、`QT_FONT_DPI` 或兼容性缩放覆盖后重启。

## 常见问题

按钮无反应：先查看顶部消息栏，已有后台任务时会阻止重复触发。

Gmail API 授权失败：检查 credentials.json、网络和 OAuth 状态，再点击显式授权。

IMAP 993 不可用：优先切换 Gmail API。

QQ SMTP 失败：确认使用 QQ 邮箱授权码，不是 QQ 登录密码。

文件选错或被修改：重新选择文件，不要只修改邮件主题。

页面数据旧：点击对应页面“刷新”，确认最后刷新时间变化。

配置保存失败：检查 `.env` 所在目录写权限，运行时旧配置不会被错误替换。
