# AgentMailBridge GUI 使用说明

AgentMailBridge 1.0.0 的用户入口只有 `AgentMailBridge.exe`。

顶部主工作区只有“收件”和“发件”。左侧已有 Gmail、QQ 账号卡片用于管理当前账号；“添加邮箱账号”只展示未来扩展和当前限制，不会修改已有账号。左侧底部只有“历史记录”“文件与数据”“设置”“关于”。

- 收件：标题右侧唯一刷新、当前 Gmail、自动收取、检查间隔、连接测试、立即收取、可编辑收件偏好、今日文件和最近日志。
- 发件：手动选择文件、快照与 Hash 保护、固定收件人、主题、进度、最近发送记录和 Agent / MCP 二级面板。
- 历史记录：以“类型、摘要、时间、状态、操作”查看收件、发件、Agent / MCP 业务记录，可按关键词或 request_id 搜索并定位关联文件；request_id、完整路径、原始状态和错误信息只在结构化详情中显示。
- 文件与数据：通过统一受管文件数据源管理收件文件、发送归档和去重后的 Agent 结果，并提供文件详情、存储概览、备份、恢复和一致性扫描。
- 设置：开机启动、界面主题、单次收取量和最大发送文件大小。
- 设置 > 高级设置：网络模式、Runtime Paths、配置迁移和脱敏高级诊断。
- 关于：版本、产品定位、仓库、LICENSE、第三方说明和本地优先说明。

Gmail API、Gmail IMAP 和 QQ SMTP 分别使用专属账号页。Gmail IMAP 密码和 QQ 授权码保存在 Windows Credential Manager，界面只显示固定掩码。替换 OAuth 客户端配置需确认，不会静默删除 token。

“当前收件偏好”默认是“仅本人邮件”。点击“编辑偏好”可选择“仅本人邮件”“当前扫描范围内全部邮件”或“自定义规则”。自定义规则只支持发件人完整邮箱或 `@domain.com`、主题关键词和“仅保存含附件的邮件”；不同分类必须同时满足，同一分类命中任一项即可。空自定义规则不能保存。该设置由 Gmail API、Gmail IMAP、手动立即收取和自动收取共用，旧布尔配置会自动无损迁移。

收件提示分为四类：成功为绿色；正常检查但无新邮件为中性提示；部分完成为黄色警告；主任务无法执行才显示红色失败。无新邮件不会增加失败统计。

今日文件只显示“文件名、大小、收取时间、操作”四列，时间使用 `HH:mm:ss`。完整路径仍保存在内部数据中，用于搜索、双击安全预览、Windows 默认程序“打开”和“复制路径”，不在主表显示。文件名列优先伸展并允许合理换行，两个操作按钮等高且垂直居中。

“文件与数据”中的收件文件直接来自 `received_files`，不从邮件历史推算大小；发送归档来自 `sent_files`，旧记录仅在安全允许目录内读取文件元数据补全大小；Agent / MCP 结果与发送记录按关联关系去重。主表不展示绝对路径，完整路径位于文件详情和复制操作。大小显示区分 `0 B`、未知的 `—` 和“文件已不存在”。数据概览卡片分别显示数据库状态、数据库大小、收件文件、发送归档、Agent 结果和备份占用。

右侧“连接健康”分别展示 Gmail 收件、QQ SMTP、Agent/MCP、凭据/OAuth、SQLite/数据目录的状态、说明和最近检查时间；“一键检查全部”刷新五项结果，异常项可直接“去处理”。

今日文件、最近日志和最近发送记录使用内部滚动，不使用分页。完整业务记录由历史记录管理，实际文件和维护操作由文件与数据管理。

关闭主窗口默认进入托盘；托盘可恢复或正常退出。开机启动默认关闭，启用后只启动主程序；内部 MCP 不显示 GUI、托盘或开机启动项。
