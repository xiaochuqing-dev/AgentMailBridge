# AgentMailBridge GUI 使用说明

AgentMailBridge 1.0.0 的用户入口只有 `AgentMailBridge.exe`。

顶部主工作区只有“收件”和“发件”。左侧已有 Gmail、QQ 账号卡片用于管理当前账号；“添加邮箱账号”只展示未来扩展和当前限制，不会修改已有账号。左侧底部只有“历史记录”“文件与数据”“设置”“关于”。

- 收件：当前 Gmail 和收件方式、自动收取、检查间隔、当前连接测试、立即收取、一键检查、今日文件和最近日志。
- 发件：手动选择文件、快照与 Hash 保护、固定收件人、主题、进度、最近发送记录和 Agent / MCP 二级面板。
- 历史记录：筛选和查看收件、发件、Agent / MCP 业务记录，可按 request_id 搜索并定位关联文件。
- 文件与数据：管理收件文件、发送归档、Agent 结果、存储概览、备份、恢复和一致性扫描。
- 设置：开机启动、界面主题、单次收取量和最大发送文件大小。
- 设置 > 高级设置：网络模式、Runtime Paths、配置迁移和脱敏高级诊断。
- 关于：版本、产品定位、仓库、LICENSE、第三方说明和本地优先说明。

Gmail API、Gmail IMAP 和 QQ SMTP 分别使用专属账号页。Gmail IMAP 密码和 QQ 授权码保存在 Windows Credential Manager，界面只显示固定掩码。替换 OAuth 客户端配置需确认，不会静默删除 token。

今日文件、最近日志和最近发送记录使用内部滚动，不使用分页。完整业务记录由历史记录管理，实际文件和维护操作由文件与数据管理。

关闭主窗口默认进入托盘；托盘可恢复或正常退出。开机启动默认关闭，启用后只启动主程序；内部 MCP 不显示 GUI、托盘或开机启动项。
