## 3. 实际发现的问题

1. 左侧“仪表盘”实际跳回基础配置，不是独立首页。
2. 基础测试连接和手动收件虽使用后台任务，但没有把被点击按钮传入统一运行态。
3. 基础操作和维护按钮横向排列，在窄窗口或高 DPI 下拥挤。
4. 文件表格写着“复制路径、预览、打开”，单击实际只复制路径。
5. 备份验证/恢复选择取消、维护报告导出取消、打开备份目录失败缺少统一反馈。
6. partial、duplicate、cancelled 缺少区别于 success/failed 的警告视觉。
7. 深色主题部分状态文字、统计卡片和 QTextEdit 使用浅色主题硬编码。
8. 账号卡片固定显示“已连接”，可能把“已配置”误解为网络连通。
9. 页面显示 gmail_api、TOKEN_EXPIRED_REFRESHABLE 等内部状态码。
10. 最终 Logo 尚未进入仓库，窗口和托盘仍使用占位图标。
11. 空表没有明确空状态，导航图标混用字符且部分字体下显示方框。

## 4. 阶段 A：真实交互审计

运行时审计覆盖 60 个 QPushButton、3 个 QCheckBox、1 个自动收件开关、4 个 QComboBox、2 个 QSpinBox、9 个表格、2 个可点击账号卡片，以及 Windows 托盘的显示、收件、刷新和退出动作。

控件行为矩阵摘要如下。

| 控件组 | 页面/入口 | 实际服务或动作 | 即时/运行中反馈 | 重复保护 | 结果与刷新 |
|---|---|---|---|---|---|
| 左侧导航、顶部标签、账号卡片 | 全局 | `select_page` | 选中态和页面切换 | 不启动任务 | 保留页面可用性 |
| 主题、最小化、最大化、关闭 | 标题栏 | 主题保存、托盘、窗口状态 | 图标/窗口立即变化 | 不适用 | 失败显示消息 |
| 自动收件、间隔、规则 | 基础配置 | 定时器与配置脏状态 | 消息栏、状态行、未保存提示 | 单定时器 | 保存后刷新配置 |
| 保存基础/高级配置 | 配置页 | 安全凭据与 `.env` 更新 | 校验、成功/失败消息 | 同步原子路径 | 刷新账号与状态 |
| Gmail 授权和三项诊断 | 配置页 | ApplicationService | 被点击按钮 running | 全局任务互斥 | 结果、错误详情、恢复按钮 |
| 手动/自动收件 | 仪表盘/基础/收件/托盘 | `service.receive` | 收取中、按钮禁用 | 手动入口同步禁用 | 文件、历史、日志、统计刷新 |
| 文件选择、复制、定位、预览 | 发件页 | 文件快照与安全预览 | 选择/取消/成功/失败消息 | 文件变化二次校验 | 元数据立即更新 |
| 发送到绑定 Gmail | 发件页 | `send_user_selected_file` | 确认、进度条、running | 任务互斥和 request_id | sent/partial/duplicate/failed，相关页刷新 |
| 日志/首页/收件/MCP 刷新 | 多页 | 统一数据收集 | 刷新中和时间戳 | 任务互斥 | 表格与状态区更新 |
| 备份、验证、恢复、扫描、导出 | 数据维护 | ApplicationService | 取消、running、完成、失败 | 任务互斥；恢复二次确认 | 维护状态和表格刷新 |
| MCP 复制与刷新 | Agent 接口 | 剪贴板、历史查询 | 复制成功、刷新中 | 任务互斥 | 调用记录刷新 |
| 表格行 | 收件/文件 | 单击复制、双击安全预览 | 消息栏或预览窗 | 路径白名单 | 不执行危险文件 |
| 托盘菜单 | Windows 托盘 | 显示、收件、刷新、退出 | 窗口/通知/消息栏 | 共用任务互斥 | 安全等待任务退出 |

所有耗时入口均有首次反馈。重复点击会显示“已有任务正在运行”，不会重复启动 SMTP、收件、备份、恢复或刷新任务。页面切换后后台结果仍通过 Qt 信号回到 GUI 线程。

## 5. 阶段 B：统一反馈与异步状态

统一结果覆盖 success、partial、failed、duplicate、cancelled、auth_required；idle 和 running 由按钮任务属性表达。partial、duplicate 和 cancelled 使用警告态，不再显示为纯成功或普通失败。按钮完成后 1.2 秒恢复常态，失败不会永久禁用。

MessageBar 增加 warning 视觉和完整文本 Tooltip。收件按钮显示“收取中…”，刷新显示“刷新中…”，发送同时显示进度条和阶段文字。取消文件、取消验证备份、取消恢复、取消导出也会返回明确消息。

## 6. 阶段 C：信息架构与主要页面

仪表盘：新增真实页面，优先展示服务健康、今日收取/保存/发送/异常、立即收取、发送文件和最近活动。

收件页：保留搜索、立即收取、刷新、双击预览；自动收件开启时所有手动入口禁用并说明原因。

发件页：保持本次明确选择、元数据、固定收件人、发送确认和文件变化校验；修正表格操作文案。

高级设置：保留 Gmail/IMAP/QQ 分区语义、凭据状态、诊断和脱敏详情；内部状态码在普通用户区域转换为可读中文。

Agent 接口：保持 stdio、固定收件人、允许目录、复制配置和调用历史，不扩展为 Agent 管理平台。

数据维护：操作区改为四列网格；恢复继续作为二次确认的危险操作；空表显示“暂无数据”。

日志与历史：刷新状态和空状态统一，表格不会因长路径改变列布局。

## 7. 阶段 D：视觉、字体、图标与 Logo

视觉继续采用紫、蓝、青体系。重点按钮使用克制渐变，普通按钮、表格和输入区保持低饱和。导航统一改用 Qt 系统图标，消除随机字符和缺字方框。

深色主题补齐状态文字、导航、统计卡片、输入框和 QTextEdit 的背景与对比度。Microsoft YaHei UI、Noto Sans SC、Segoe UI 与 Qt 高 DPI 策略保持不变。

最终 Logo 来自用户桌面 `logo素材.jpg`。仓库保留原始 JPG，并生成透明 512 PNG、16/24/32/48/64/128/256 PNG 和多尺寸 ICO。已接入标题栏、窗口图标、Windows 任务栏、系统托盘和关于页。小尺寸版本采用紧凑裁切、透明背景、Lanczos 缩放和轻量轮廓锐化，没有改动原 Logo 结构。

## 8. 阶段 E：真实 Windows 路径与视觉 QA

使用隔离数据完整启动 GUI，逐页切换仪表盘、收件、发件、高级设置、Agent 接口、数据维护和日志。验证刷新、任务状态、页面切换、空状态、浅色/深色主题和关闭路径。

Windows 原生图形后端报告 `logical_dpi=96.0`、`dpr=1.50`，7 个主要页面均完成真实系统截图 QA。另使用 Qt 离屏比例 1.00、1.25、1.50 生成 21 张布局截图，并补充 2 张深色主题截图。未发现文字截断、卡片溢出、表格不可读或页面重叠。截图目录 `artifacts/gui-qa-part1/` 已被 Git 忽略。
