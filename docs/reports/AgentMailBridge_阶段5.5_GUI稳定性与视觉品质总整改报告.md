# AgentMailBridge 阶段 5.5 GUI 稳定性与视觉品质总整改报告

## 1. 执行摘要

本轮在不重写 Gmail、QQ SMTP、MCP、SQLite 和 ApplicationService 主链路的前提下，完成发件防误选、异步按钮反馈、页面刷新、配置回滚、友好错误、脱敏诊断、字体与 DPI、状态恢复和 GUI 回归整改。自动化测试 213 项全部通过。阶段 A、B、C 通过；阶段 D、E、F 为有条件通过，原因是当前机器系统缩放只有 100%，未做多小时稳定性运行，也未再次发送真实验收邮件。

## 2. 开始前项目状态

开始分支为 `codex/email-ui-fixes`，工作区干净。已有 Gmail API 收件、QQ SMTP 发件、MCP、托盘、自动收件、单实例和 PySide6 GUI。既有提交已修复诊断按钮联动禁用和日志顺序，但发件页仍缺少完整文件确认、变化检测、完成后清空和明确进度。

整改前 Git 存档：`32e232c chore: 阶段5.5整改前存档`。

## 3. 用户发现的问题

已确认并处理四类高优先级问题：发送按钮缺少即时反馈；旧待发送文件可能误发；高级设置授权和诊断反馈不足；首页和日志刷新状态不清晰。字体小、控件层次不足和高 DPI 下按钮文字截断也纳入整改。

## 4. 严重问题修复结果

发送入口改为默认禁用，必须本次明确选择文件。后台任务立即显示运行状态并阻止重复触发。四个高级设置按钮调用真实 ApplicationService。首页、收件、日志和 MCP 页面均增加后台刷新入口和最后刷新时间。

## 5. 发件按钮反馈整改

点击确认后，发送按钮立即禁用并改为“正在发送…”，显示无限进度条和当前步骤说明。完成后显示发送成功、SMTP 失败、重复请求或“邮件已发送但本地归档失败”的不同结果。按钮提供 normal、hover、pressed、disabled、running、success、error 样式。

## 6. 待发送文件状态整改

新增本次选择快照，记录解析后的路径、大小和纳秒级修改时间。界面展示文件名、大小、类型、最后修改时间和完整路径提示，并提供复制路径、打开所在文件夹和安全预览。用户确认前检查一次，后台真正发送前再检查一次。文件被删除、移动或修改时阻止发送。成功、重复或已发送但归档失败后清空文件和主题。页面和程序重启均不恢复待发送文件。

## 7. 四个诊断与授权按钮整改

Gmail API 显式授权、诊断 IMAP、诊断 Gmail API、诊断 QQ SMTP 都绑定真实服务调用。执行时只禁用当前按钮；其他按钮保持可见可用，但统一任务互斥会阻止并发后台操作。结果回到 GUI 主线程显示。

真实只读诊断结果：Gmail API 通过；QQ SMTP 连接与认证通过；Gmail IMAP direct 和 SOCKS5 均在 TLS 阶段失败，继续推荐 Gmail API。诊断过程未发送邮件、未读取邮件正文。

## 8. 日志刷新整改

首页最近日志和完整日志统一按时间倒序。首页、日志页、收件页和 MCP 页有刷新入口；刷新在线程池执行，避免数据库和文件查询阻塞 GUI。完成后显示最后刷新时间，失败时显示友好原因和脱敏详情。

## 9. 全界面按钮审计结果

运行时审计发现 47 个 QPushButton，47 个均绑定 clicked 信号，无未绑定按钮。导航、配置、收件、发件、预览、路径操作、授权、诊断、刷新、MCP、主题、窗口和托盘入口均有可见反馈。高级设置从单行按钮改为两行网格，解决 125% 和 150% 缩放文字截断。

## 10. 统一异步反馈体系

继续使用单线程 QThreadPool 和统一 `_run_task`。状态覆盖 idle、running、success、failed、partial、cancelled、duplicate、auth_required。任务运行时保存按钮原文、显示工作文案、禁用当前按钮、设置状态属性；结束后恢复按钮并短暂展示成功或错误颜色。异常统一转换为 ServiceResult 返回 GUI。

## 11. 页面刷新体系

收件、发件、授权、诊断完成后自动刷新配置、今日文件、日志、历史、MCP 和右侧统计。手动刷新使用同一数据收集入口，并禁止重复刷新任务。每次成功刷新记录时间。

## 12. 配置页面改进

新增配置脏状态提示和离页确认。邮箱格式和 QQ 邮箱/授权码成对校验继续生效。密码和授权码支持临时显示/隐藏。高级设置先处理开机启动，再原子写入 `.env`；写入失败时回滚开机启动状态，运行时配置保持旧值。

## 13. 错误映射

OAuth、Gmail API、IMAP 993、QQ SMTP、文件变化、路径越界、收件繁忙和刷新失败都有面向用户的说明与建议。技术详情单独保存并脱敏，不直接向普通消息栏输出 traceback。

## 14. 脱敏诊断报告

ApplicationService 新增诊断报告导出，包含应用、Windows、Python、PySide6、后端、OAuth、配置完整性、SQLite、日志轮转和错误计数。排除密码、授权码、token、credentials 内容、邮件正文、附件内容和私人绝对路径。已有目标文件不会被覆盖。

## 15. 字体模糊根因

原样式大量使用 10px 和 11px 小字，字体候选优先加载可变 Noto 字体，且没有显式高 DPI rounding policy。未发现 QGraphicsDropShadowEffect、父控件透明度或整体缩放导致的文字栅格化。主要风险是字号偏小、字体回退和外部 QT 缩放变量叠加。

## 16. DPI 与字体修复

优先使用 Windows 原生 Microsoft YaHei UI，回退 Noto Sans SC 和 Segoe UI，并启用抗锯齿策略。Qt 6 使用 PassThrough 高 DPI 比例，不设置 QT_FONT_DPI 或额外重复缩放。表单、按钮、表格和标题字号提高到清晰整数值。

## 17. 图标和像素对齐

现有图标继续使用 Qt 绘制和文本符号，没有放大低分辨率 PNG。输入框、按钮和表格使用整数高度、整数边框和统一基线。未引入文字父控件阴影或透明缩放。

## 18. 炫彩视觉升级

主操作按钮保留紫色渐变；发送加载条使用紫色到蓝色渐变；成功和失败状态使用克制的绿、红色；统计卡片继续使用浅紫、浅绿、浅蓝和浅红。未增加大面积彩虹背景或持续闪烁动画。

## 19. 安全与状态恢复

窗口几何、上次页面、主题、自动收件和正常退出标记可恢复。待发送文件、未确认任务、临时错误和敏感明文不持久化。当前配置摘要隐藏完整路径并脱敏邮箱。导航跨页面选中态已修复。

## 20. 长时间与资源释放

任务线程池最大并发为 1；重复操作被拦截。正常退出等待任务、停止定时器、关闭 SQLite、隐藏托盘并释放单实例锁。日志保持 2 MB、5 个备份轮转。自动化覆盖多次任务、刷新和窗口关闭；本轮未进行多小时内存曲线测试，因此长期稳定性为有条件通过。
