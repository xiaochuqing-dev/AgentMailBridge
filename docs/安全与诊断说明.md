# 安全与诊断说明

## 凭据和 OAuth

Gmail IMAP 应用专用密码和 QQ SMTP 授权码保存在当前用户 Windows Credential Manager。普通配置保存层拒绝非空 `GMAIL_APP_PASSWORD` 和 `QQ_AUTH_CODE`；GUI 不回显旧值，写入后必须回读验证，失败时恢复旧凭据。

Gmail OAuth `credentials.json` 和 `token.json` 位于 `%LOCALAPPDATA%\AgentMailBridge\OAuth`，升级和普通卸载保留。GUI 只复制用户明确选择且结构有效的 credentials.json。Gmail scope 必须且只能是 `https://www.googleapis.com/auth/gmail.readonly`。

## 路径和数据

安装目录只放 EXE、DLL、Qt 和只读资源。配置、OAuth、数据、SQLite、日志、邮件、附件和备份在当前用户可写目录。自定义 DATA_ROOT 继续支持，相对路径基于配置文件目录，不依赖当前工作目录。

普通卸载不删除用户数据、OAuth 或 Credential Manager 凭据。彻底清理必须作为独立高风险操作设计，本版本未提供静默清理。

## 诊断

GUI 提供 Gmail API、Gmail IMAP、QQ SMTP、网络、数据库和维护诊断。源码 CLI 还提供：

```powershell
python -m agent_mail_bridge diagnose-gmail-api
python -m agent_mail_bridge diagnose-gmail
python -m agent_mail_bridge diagnose-qq-smtp
python -m agent_mail_bridge diagnose-network
python -m agent_mail_bridge credential-status
```

安装版发布验收可使用 `AgentMailBridge.exe --diagnose-gmail-api`、`--diagnose-qq-smtp` 和 `--packaged-self-test`，这些模式以退出码报告结果，不输出秘密。

## 发布安全检查

构建只显式收集程序资源、LICENSE 和第三方说明，不收集仓库 `.env` 或 secrets。`verify_build.ps1` 和 `secret_scan.py` 检查 dist、portable ZIP 和 release 文件名/内容来源；Defender 对 dist 与 release 的自定义扫描未产生新增检测。

当前 EXE 与安装器未签名，Authenticode 状态为 NotSigned，SmartScreen 可能提示未知发布者。不要绕过提示或伪造签名。
