# AgentMailBridge 安全与诊断说明

## 发件边界

收件人固定为配置中的 OWNER_GMAIL，GUI 和 MCP 都不能临时指定其他收件人。发送文件必须位于 DATA_ROOT 或 ALLOWED_SEND_ROOTS。GUI 每次启动不恢复待发送文件，页面切换也不会从历史记录偷偷恢复旧文件。

选择文件后记录绝对路径、大小和高精度修改时间。用户确认前检查一次，后台连接 SMTP 前再检查一次。文件被删除、移动、修改或大小变化时阻止发送。发送核心继续由 ApplicationService 调用现有 QQ SMTP、request_id 幂等和本地归档逻辑。

## 预览与打开

文本和常见图片使用应用内只读预览。exe、bat、cmd、ps1、sh、msi 以及其他不在安全预览列表中的类型不会直接执行，只调用资源管理器定位文件。所有预览路径都必须位于 DATA_ROOT。

## 凭据与配置

Gmail 应用专用密码、QQ 授权码、OAuth token 和 credentials.json 不写入代码或报告。密码框默认隐藏，只能由用户临时切换显示。当前配置摘要隐藏完整本地路径，并对邮箱和密钥脱敏。

`.env`、credentials.json、token.json、secrets、日志、用户数据库、测试输出、GUI 截图和诊断报告均由 `.gitignore` 排除。新增配置项必须只在 `.env.example` 提供无秘密示例。

## 脱敏诊断报告

诊断报告只包含应用版本、Windows 版本、Python 和 PySide6 版本、收件后端、配置是否完整、OAuth 状态、SQLite 状态、日志轮转数量和错误事件数量。

报告不包含完整邮箱、密码、授权码、token、credentials 内容、邮件正文、附件内容或私人绝对路径。最近错误只记录时间和事件类型，详细消息隐藏。为避免意外覆盖证据，目标文件已存在时导出失败并要求新文件名。

## 日志与错误详情

GUI 的错误详情会替换当前 Gmail 密钥、QQ 授权码、邮箱、token 路径、credentials 路径和数据目录。普通消息优先显示用户可理解的原因和下一步建议，不直接展示 traceback。

日志采用 2 MB 单文件、保留 5 个备份的轮转策略。业务日志不得记录邮件正文、附件内容或凭据。

## MCP 边界

MCP 使用本机 stdio，不监听网络端口。Agent 只能提交允许目录内的文件，不能读取邮箱、删除邮件、修改邮箱配置、指定任意收件人或读取凭据。调用写入 mcp_calls 审计，并复用 ApplicationService 和发送幂等机制。

## 异常退出与恢复

程序通过 QSettings 保存窗口几何、上次页面和正常退出标记。正常退出会等待线程池、关闭 SQLite 连接、隐藏托盘并释放单实例锁。异常退出后只恢复安全界面偏好，不恢复临时错误、未确认任务或待发送文件。

## 当前限制

真实 Gmail 和 QQ SMTP 验收会产生外部网络访问和真实邮件，只能在用户明确选定测试文件时进行。自动化测试始终隔离真实 `.env`、OAuth 文件、邮箱、数据库和用户文件。本轮阶段 5.5 未发送真实验收邮件，外部链路结果沿用阶段 3 至 5 记录。
