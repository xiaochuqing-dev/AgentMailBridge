# AgentMailBridge 安全与诊断说明

## 发件边界

收件人固定为配置中的 OWNER_GMAIL，GUI 和 MCP 都不能临时指定其他收件人。GUI 用户可通过系统选择器明确选择任意位置的普通文件；CLI、MCP 和自动流程仍必须位于 DATA_ROOT 或 ALLOWED_SEND_ROOTS。GUI 每次启动不恢复待发送文件，页面切换也不会从历史记录恢复旧文件。

选择文件后记录绝对路径、大小、高精度修改时间和 SHA-256。用户确认前检查一次，后台连接 SMTP 前再检查一次，并在 DATA_ROOT/send/staging 创建受控快照，核对大小和 Hash。数据库公开历史不保存外部原始绝对路径；SMTP 和 sent 归档使用快照，request_id 继续保证幂等。

## 预览与打开

文本和常见图片使用应用内只读预览。exe、bat、cmd、ps1、sh、msi 等危险类型不会发送或执行。GUI 本次明确选择的全局文件可安全预览；数据库与收件历史预览仍限制在 DATA_ROOT。

## 凭据与配置

Gmail 应用专用密码和 QQ 授权码由 Windows Credential Manager 按当前 Windows 用户保存，GUI、CLI 和 ApplicationService 通过统一入口读取。密码框不从系统存储回显旧值。旧 `.env` 迁移先写入并回读验证，成功才清空对应明文，失败不破坏旧配置。

OAuth credentials.json 和 token.json 继续存放在 secrets 路径，保持 Gmail readonly scope，不进入 Windows Credential Manager、数据库备份、日志或诊断报告。删除 token 后需要重新授权。

`.env`、credentials.json、token.json、secrets、日志、用户数据库、测试输出、GUI 截图和诊断报告均由 `.gitignore` 排除。新增配置项必须只在 `.env.example` 提供无秘密示例。

## 脱敏诊断报告

诊断报告只包含应用版本、Windows 版本、Python 和 PySide6 版本、收件后端、配置是否完整、OAuth 状态、SQLite 状态、日志轮转数量和错误事件数量。

报告不包含完整邮箱、密码、授权码、token、credentials 内容、邮件正文、附件内容或私人绝对路径。最近错误只记录时间和事件类型，详细消息隐藏。为避免意外覆盖证据，目标文件已存在时导出失败并要求新文件名。

## 日志与错误详情

GUI 的错误详情会替换当前 Gmail 密钥、QQ 授权码、邮箱、token 路径、credentials 路径和数据目录。普通消息优先显示用户可理解的原因和下一步建议，不直接展示 traceback。

日志采用 2 MB 单文件、保留 5 个备份的轮转策略。业务日志不得记录邮件正文、附件内容或凭据。

GUI 产品化整改只调整界面组织、反馈、主题、图标和品牌资源，不改变固定收件人、路径白名单、快照校验、request_id 幂等、OAuth readonly scope、Windows Credential Manager 或数据库维护边界。截图 QA 使用隔离数据并由 Git 忽略。

## MCP 边界

MCP 使用本机 stdio，不监听网络端口。Agent 只能提交允许目录内的文件，不能读取邮箱、删除邮件、修改邮箱配置、指定任意收件人或读取凭据。调用写入 mcp_calls 审计，并复用 ApplicationService 和发送幂等机制。

## 数据维护边界

数据库备份使用 SQLite 在线备份并执行 integrity_check 和 SHA-256 清单校验。恢复前自动备份当前数据库，失败时回滚；received、send 和 sent 用户文件不随数据库恢复覆盖。

一致性扫描默认只报告数据库引用缺失、孤立文件、Hash 异常、越界路径、暂存残留和无法访问项，不自动删除、移动、合并或重建数据。维护报告不包含正文、附件内容、凭据、token 或完整私人路径。

## 异常退出与恢复

程序通过 QSettings 保存窗口几何、上次页面和正常退出标记。正常退出会等待线程池、关闭 SQLite 连接、隐藏托盘并释放单实例锁。异常退出后只恢复安全界面偏好，不恢复临时错误、未确认任务或待发送文件。

## 当前限制

真实 Gmail API 与 QQ SMTP 只读诊断已通过，但本专项未触发真实收件或发信。自动化测试和性能基准隔离真实 `.env`、Windows 凭据、OAuth 文件、邮箱、数据库和用户文件。多显示器切换、休眠唤醒和连续数日运行仍需在日常环境观察。
