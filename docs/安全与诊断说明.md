# 安全与诊断说明

Gmail IMAP 应用专用密码和 QQ SMTP 授权码保存在当前用户 Windows Credential Manager。界面使用固定 16 位视觉掩码，不按真实 secret 长度生成掩码，不回显旧值；新值保存失败时保留旧凭据。

Gmail OAuth 位于 `%LOCALAPPDATA%\AgentMailBridge\OAuth`。GUI 只导入用户明确选择且结构有效的 `credentials.json`，替换需确认，不静默删除 token。Gmail scope 必须且只能是 `https://www.googleapis.com/auth/gmail.readonly`。

安装目录只放程序与只读资源。配置、OAuth、数据、SQLite、日志、邮件、附件和备份位于当前用户可写目录。普通卸载不删除用户数据、OAuth 或 Credential Manager 凭据。

诊断入口：

- Gmail API、Gmail IMAP、QQ SMTP 单项测试位于对应账号页。
- 收件页右侧“连接健康”以五个独立状态项聚合 Gmail、QQ SMTP、MCP、凭据/OAuth 和 SQLite/数据目录，记录最近检查时间并提供“去处理”。
- 收件页“管理日志”支持搜索、级别、时间范围、详情、脱敏诊断导出和日志目录。
- 设置 > 高级设置保留网络、路径、迁移、全部连接诊断、脱敏报告和最近错误。
- 文件与数据 > 数据维护负责备份、验证、恢复、一致性扫描和维护报告；恢复必须确认并先备份当前数据库。

源码 CLI 继续提供 Gmail、QQ、网络、凭据和数据诊断。构建前必须执行秘密扫描；`.env`、credentials.json、token.json、数据库、日志、邮件和附件不得进入 Git、dist、release 或报告。

收件结果的安全语义固定为 success、no_changes、partial、failed。正常无新邮件使用 no_changes，不写 ERROR、不增加失败统计；单项处理失败但已保存部分内容使用 partial 和 WARNING；只有认证、网络、连接或主流程不可执行等真实故障才使用 failed 和 ERROR。

当前 EXE 与安装器未签名时，Authenticode 状态为 NotSigned，SmartScreen 可能提示未知发布者。不得绕过提示或伪造签名。
