# AgentMailBridge 第一批次实现报告

## 完成内容

本次在保留 IMAP、Gmail API、QQ SMTP 和原 CLI 的基础上，完成了统一应用服务与最小 GUI 骨架。

新增核心模块：

- application_service.py：CLI、GUI 和未来 MCP 的统一业务入口。
- models.py：success、partial、failed、auth_required、duplicate、cancelled 等结构化结果。
- mail_common.py：统一地址解析、Gmail 别名、Message-ID 和附件安全状态。
- mail_processing.py：两个收件后端共用的可信来源判断、去重、保存和数据库登记流程。
- gui.py：tkinter 最小 GUI 骨架，后台执行长任务并阻止重复点击。

现有模块整改：

- database.py：启用 WAL、5000 毫秒 busy timeout、短事务、邮件唯一约束、发送 request_id 唯一约束和幂等状态更新。
- mail_receive.py、gmail_api_receive.py：后端只负责读取和转换，后续进入统一处理流程；普通收件不再自动打开 OAuth 浏览器。
- mail_send.py：底层 SMTP 连接实际使用超时配置，区分连接、认证、发送错误，并支持 attempt_created、sent、failed、sent_archive_failed、duplicate。
- gmail_api_auth.py：增加 OAuth 互斥和 READY、AUTH_REQUIRED、TOKEN_EXPIRED_REFRESHABLE、TOKEN_INVALID、SCOPE_MISMATCH、CREDENTIALS_MISSING 状态。
- security.py、file_index.py：限制 DATA_ROOT 和 ALLOWED_SEND_ROOTS，拦截路径穿越、Windows 保留名及旧数据库越界路径；危险附件只保存并标记。
- cli.py：收件、发件、列表、扫描、授权和三类诊断通过 ApplicationService 调用。
- config.py、.env.example：增加允许发送目录配置，并强制 Gmail API 只能使用 gmail.readonly。
- README.md、.gitignore、pyproject.toml：更新架构、安全、GUI、启动命令、忽略规则和 GUI 命令入口。

## 一致性与恢复策略

跨后端优先使用归一化 RFC Message-ID 去重；缺少 Message-ID 时按发件人、收件人、主题、日期、正文和附件 hash 生成稳定键。

单封邮件通过 BEGIN IMMEDIATE 事务完成唯一占位、确定性文件写入、文件记录和 saved 状态更新。文件或数据库失败会回滚数据库，不会留下“已完成”假状态；确定性路径保证重试不产生第二套文件。

发送先登记 attempt_created。SMTP 成功后立即登记 sent，再做 sent 归档；归档失败记录 sent_archive_failed，同一 request_id 重试不会再次发信。SMTP 明确失败后，同一 request_id 可以安全重试。

## 测试结果

自动化测试完全禁用项目真实 .env，credentials.json、token.json 和 DATA_ROOT 均指向 pytest 临时目录。

最终结果：179 passed，耗时 1.37 秒。

重点覆盖：

- IMAP 与 Gmail API 跨后端只保存一次。
- 两个并发收件只生成一条记录和一套文件。
- 文件写入失败时数据库不出现完成记录。
- 多层 multipart、纯文本、HTML 和内联附件结果一致。
- 地址显示名、多收件人、Cc、大小写、加号地址和 Gmail 别名。
- OAuth 缺凭据、缺授权、token 和 scope 状态。
- SMTP 超时传到底层、归档失败、重复请求和失败后重试。
- GUI 重复点击不会启动第二个任务。
- DATA_ROOT 外发送路径和旧数据库越界路径被拒绝。
- 原 164 项测试继续通过，总数增加到 179 项。

## 验证命令

```powershell
python -m compileall -q agent_mail_bridge
python -m pytest -q
python -m agent_mail_bridge --help
python -m agent_mail_bridge init
python -m agent_mail_bridge receive
python -m agent_mail_bridge.gui
```

发件文件必须位于 DATA_ROOT 或 ALLOWED_SEND_ROOTS：

```powershell
python -m agent_mail_bridge send --file AgentMailBridgeData\send\result.txt --request-id manual-001
```

诊断命令：

```powershell
python -m agent_mail_bridge diagnose-gmail
python -m agent_mail_bridge diagnose-gmail-api
python -m agent_mail_bridge diagnose-qq-smtp
```

## 手动验收建议

1. 运行 GUI，确认显示收件后端、OAuth 和 QQ SMTP 配置状态。
2. 连续点击两次手动收取，确认第二次不会启动。
3. 用同一封 Gmail 自发自收邮件分别经过 IMAP 和 Gmail API 收取，确认只新增一条记录。
4. 从 DATA_ROOT 内选择测试文件发送，重复使用相同 request_id，确认只发送一次。
5. 断网后分别运行三类诊断，确认显示具体连接或认证错误。
